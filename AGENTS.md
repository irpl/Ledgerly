# Project: personal finance tracker

Spec lives in `docs/finance-app-plan.md` (v2). Money is stored as signed BigInt minor units — out negative, in positive.

## Dev setup

- Local DB: Prisma Postgres local server (`npx prisma dev --detach`), NOT Docker — Docker Desktop's WSL bootstrap is broken on this machine. `docker-compose.yml` exists as an alternative if Docker is fixed. If ports change after a restart, re-run `npx prisma dev --detach` and update `DATABASE_URL`/`SHADOW_DATABASE_URL` in `.env` (see `npx prisma dev ls`).
- Schema changes: `npx prisma db push` for local dev (`migrate dev` fails against the PGlite dev server with P1017). For deployable migrations, generate SQL offline: `npx prisma migrate diff --from-schema-datasource <old> --to-schema prisma/schema.prisma --script` into `prisma/migrations/`.
- Seed (idempotent, creates login user from ADMIN_EMAIL/ADMIN_PASSWORD): `npx prisma db seed`.
- `src/lib/account-shared.ts` is client-safe; `src/lib/accounts.ts` and `src/lib/prisma.ts` are server-only (importing them from a client component breaks the browser bundle on pg/node builtins).
- Route protection: `src/proxy.ts` (Next 16 proxy, JWT check only, no DB); credentials auth in `src/lib/auth.ts`.

## Design system

Generated by github.com/nextlevelbuilder/ui-ux-pro-max-skill for "personal finance tracker": dark-only OLED style — background #0F172A, primary #1E40AF (trust blue), accent #059669 (profit green), Fira Sans body / Fira Code for amounts. Tokens live in `src/app/globals.css` as `@utility` classes (`card`, `input`, `label`, `btn-primary`, `btn-ghost`, `btn-danger`, `amount`). Rules: Lucide SVG icons only (no emoji in UI chrome), cursor-pointer + visible focus ring on interactive elements, 150–300ms transitions, `prefers-reduced-motion` respected. Tailwind v4: custom classes used with `@apply` must be `@utility`; important modifier is trailing (`p-0!`).

## Phase 7 notes

- PWA: `src/app/manifest.ts`, hand-written `public/sw.js` (bump VERSION when changing shell assets), icons generated by `node scripts/generate-icons.mjs`. SW registers in production builds only.
- CSV export: `GET /api/export?entity=...` (see /settings for the entity list).
- Deploy: `Dockerfile` runs `prisma migrate deploy` + idempotent seed on boot; steps in `docs/deploy-coolify.md`; email setup in `docs/email-ingestion.md`. Set `AUTH_TRUST_HOST=true` behind a proxy.

## Tests

`npm test` (vitest). Unit tests cover money/normalization/liability/parser/period/validation logic; integration tests run against a dedicated `jan_test` database (auto-synced by tests/global-setup.ts via `prisma db push`; created once with CREATE DATABASE jan_test in the jan-db container). Webhook tests hit the live dev server on :3000 and self-skip when it is down. Integration files share one DB, so vitest runs files serially (fileParallelism: false).
